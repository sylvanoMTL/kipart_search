## Successes & Strengths

1. **`SourceConfigManager` in core/ with clean separation** — Config persistence (config.json), credential storage (keyring), and environment variable overrides all in one core module with zero GUI dependencies. The `SOURCE_REGISTRY` pattern makes adding new sources declarative.

2. **Connection test as QThread worker** — `ConnectionTestWorker` runs API validation in a background thread, keeping the dialog responsive. Follows the established worker pattern from Epic 1's `SearchWorker`.

3. **Welcome dialog reuses existing components** — The JLCPCB download path reuses `DownloadWorker` from Epic 4. The API configuration path opens the same `SourcePreferencesDialog` from Story 6.1. No duplicated functionality.

4. **Code review fix rate improved** — Only 1/2 stories needed review fixes (Story 6.1). Story 6.2 passed without code changes. This is the best ratio since Story 4.1.

5. **`conftest.py` autouse fixture** — Story 6.2 added a project-wide `conftest.py` that patches `_check_welcome` to prevent modal dialog blocking during tests. This solves the systemic problem of modal dialogs interfering with test execution.

6. **Preferences toolbar button enabled** — The disabled "Preferences" placeholder from Epic 1 (Story 1.2) is now fully functional. A 6-epic-long deferred feature finally delivered.

---

## Challenges & Growth Areas

1. **Pre-existing PySide6 segfault** — Story 6.2 noted a pre-existing segfault in tests when creating multiple MainWindow instances in a single test process on Windows. This is a Qt object lifecycle issue, not a code bug, but it's affecting test reliability.

2. **Modal dialog testing** — Both stories create modal dialogs (SourcePreferencesDialog, WelcomeDialog). Modal dialogs block the event loop, making them inherently harder to test. The `conftest.py` patching approach is a workaround, not a solution.

3. **`isVisible()` vs `isHidden()` appeared again** — Story 6.1 noted "Fixed 2 GUI test failures: `isVisible()` returns False when dialog not shown; switched to `isHidden()` checks." Sixth epic in a row.

4. **Config migration path** — The transition from hardcoded JLCPCB init (`_init_jlcpcb_source()`) to config-driven init (`_init_sources_from_config()`) was clean in Story 6.1, but the ad-hoc first-run prompt removal in Story 6.2 shows that init logic had accumulated technical debt.

## Key Insights & Learnings

1. **Config-driven source initialization** — Moving from hardcoded source setup to `SourceConfigManager` + `SOURCE_REGISTRY` makes the app extensible. Adding DigiKey or Mouser only requires a registry entry and adapter class.

2. **Welcome dialog as a funnel** — Three clear options (download, configure, skip) with a bold "Preferences" button emphasis after skip ensures users aren't stuck. The skip-but-nudge pattern is good UX.

3. **conftest.py for modal dialog isolation** — Autouse fixtures that patch modal dialog triggers prevent test hangs. This should be the standard approach for any new modal dialog.

4. **Keyring + env var + config.json layering** — Three credential sources with clear priority (env var > keyring > config) gives flexibility for both interactive and CI use cases.
